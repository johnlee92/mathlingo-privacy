어린이와 보호자를 위한 요약

계정 연결은 선택 사항입니다. 연결하지 않아도 이 기기에서 학습할 수 있습니다.

이메일과 비밀번호로 계정을 연결하면 학습 프로필과 기록을 서버에 저장해 다른 기기에서도 이어서 학습할 수 있습니다. 가입 과정에서 이메일 소유권이나 보호자 관계를 별도로 인증하지 않습니다. 오프라인 학습을 위해 같은 정보의 기기 내 캐시도 사용하며, 광고를 위해 개인정보를 판매하거나 공유하지 않습니다.

2. 처리하는 정보, 목적과 보유 기간

구분정보목적보유 기간
계정과 인증 계정 이메일, Supabase가 해시 처리한 비밀번호, Supabase 계정 ID, 로그인 세션, 인증 요청 시각, IP 주소와 기기·앱·보안 로그 등 자동 생성 정보. 앱은 원문 비밀번호를 저장하지 않습니다. 비밀번호 기반 가입·로그인, 계정 식별, 로그인 유지, 부정 사용 방지, 계정 삭제 전 재인증 계정 삭제 시 활성 계정·세션 원본을 삭제합니다. 프로젝트 데이터베이스의 인증 감사 로그 저장은 비활성화되어 있으며, 외부 보안 로그·복구 사본은 Supabase의 요금제·보안 정책에 따른 기간 동안 제한적으로 남을 수 있습니다.
서버 학습 정보 학습자 표시 이름·사용자 이름·아바타 글자와 색상·가입일, 선택 학년·코스·레슨·유닛과 진도, 별·XP·정확도·콤보·시도 횟수·완료 시각, 오답과 복습 기록, 젬·하트·연속 학습·프리즈·XP 부스트, 일일 활동·퀘스트·배지·리그, 온보딩 여부, 일일 목표, 동기화용 임의 기기 ID·변경 ID·리비전과 관련 시각 학습 상태 저장, 다른 기기와 동기화, 충돌·중복 저장 방지, 학습 기능 제공 계정 삭제 시까지
게스트 기록 병합 영수증 게스트 기록 묶음마다 무작위로 만든 UUID, 병합된 계정 ID와 리비전, 생성 시각. 이메일·프로필·학습 내용은 포함하지 않습니다. 앱 백업 복원이나 다른 계정에서 같은 게스트 기록과 보상을 다시 가져오는 부정·중복 처리를 방지 계정 삭제 전에는 계정 ID와 연결됩니다. 계정 삭제 시 계정 ID를 즉시 분리하고, 나머지 UUID·리비전·생성 시각은 계정과 연결할 수 없는 1회 사용 방지 표식으로 서비스 운영 기간 동안 보관합니다.
기기 내 오프라인 캐시 게스트·계정별 학습 스냅샷 전체(프로필 포함), 대기 중인 변경·리비전·임의 기기 ID·변경 ID·순번과 관련 시각 등 동기화 메타데이터, 현재 기기의 Keychain에 저장되는 로그인 세션 오프라인 학습, 앱 재실행과 재로그인 뒤 이어하기, 서버 동기화 충돌 방지 로그아웃 뒤에도 재로그인을 위해 계정별 학습 캐시가 남을 수 있습니다. 로그인 세션은 로그아웃·계정 삭제 때 제거합니다. iOS에서는 앱 삭제 뒤 Keychain 항목이 남더라도 재설치 시 재사용하지 않고 정리하며 설치 식별 표시는 백업하지 않습니다. macOS에서는 앱을 휴지통으로 옮겨도 Keychain과 Application Support 데이터가 남을 수 있으므로, 세션 제거가 필요하면 먼저 앱에서 로그아웃하거나 계정을 삭제해야 합니다.
기기 안에만 저장 진행 중인 레슨의 현재 문제·입력 상태, 리마인더·사운드·햅틱·동작 줄이기와 안내 닫기 설정 중단한 학습 이어하기와 기기별 사용 환경 제공 서버에는 동기화하지 않습니다. iOS에서는 앱 삭제 시 제거됩니다. macOS에서는 앱을 휴지통으로 옮기는 것만으로 설정과 Application Support 데이터가 모두 제거되지 않을 수 있습니다.

3. 선택 동의와 만 14세 미만 아동

계정 연결과 서버 동기화는 선택 사항입니다. 동의하지 않으면 계정 연결과 다른 기기 동기화를 이용할 수 없지만, 게스트 모드로 이 기기에서 학습할 수 있습니다.

만 14세 미만 아동은 법정대리인과 개인정보 처리 내용을 함께 확인하고 동의한 경우에만 계정을 연결해야 합니다. 앱은 가입 전에 보호자와 확인했다는 확인란 선택을 요구하지만, 입력한 이메일의 소유권, 동의 여부·시각·처리방침 버전 또는 아동과의 관계를 서버에서 별도로 저장하거나 인증하지 않습니다. 법정대리인은 아래 문의처를 통해 아동의 개인정보 열람·정정·삭제·처리정지 및 동의 철회를 요청할 수 있습니다.

4. 처리위탁과 서비스 인프라

수탁자업무처리 위치와 기간
Supabase, Inc. 및 그 인프라 제공자 비밀번호 기반 계정 인증, 계정·데이터베이스·파일 저장, 보안·장애 대응 주 데이터베이스는 AWS 서울 리전(ap-northeast-2)에 위치합니다. 서비스 운영·지원 과정에서 Supabase 및 공개된 하위처리자가 국외에서 제한적으로 처리할 수 있으며, 계정 또는 위탁 계약 종료 시까지 처리합니다.
GitHub, Inc. 개인정보 처리방침 웹페이지 호스팅과 보안 페이지 방문 시 IP 주소와 요청 로그가 GitHub의 운영 인프라에서 보안 목적으로 처리되며, GitHub의 서비스·보안 정책에 따른 기간 동안 보관될 수 있습니다.

시행일 현재 가입·로그인용 이메일 발송과 별도 SMTP는 사용하지 않습니다. 따라서 이메일 소유권 확인과 이메일 기반 비밀번호 재설정 기능은 제공되지 않습니다. 향후 메일 발송을 활성화하면 수탁자와 처리 현황을 이 처리방침에 반영합니다.

Supabase의 최신 처리 조건과 하위처리자 정보는 Supabase Data Processing Addendum에서 확인할 수 있습니다.

5. 제3자 제공과 광고

MathLingo 운영자는 이용자의 개인정보를 판매하지 않으며 맞춤형 광고에 사용하지 않습니다. 법령상 의무가 있거나 이용자가 별도로 동의한 경우를 제외하고, 위 처리위탁 목적 밖의 제3자에게 제공하지 않습니다.

6. 삭제와 권리 행사

  • 로그아웃하면 해당 기기의 Keychain 로그인 세션을 제거하고 게스트 화면으로 전환합니다. 서버 계정·학습 정보와 이 기기의 계정별 오프라인 캐시·동기화 메타데이터는 유지되며, 같은 계정으로 다시 인증하기 전에는 표시하지 않습니다.
  • 로그인 세션은 현재 기기에 묶어 저장하며 다른 기기로 이전하지 않습니다. iOS에서는 앱 삭제 뒤 Keychain 항목이 남더라도 재설치 때 이전 세션을 자동으로 복원하지 않습니다. macOS에서는 앱을 휴지통으로 옮겨도 Keychain과 Application Support 데이터가 남을 수 있으므로 세션을 지우려면 먼저 앱에서 로그아웃하거나 계정을 삭제해야 합니다.
  • 앱의 계정 삭제에서 현재 비밀번호로 다시 로그인하면 Supabase 계정과 연결된 서버 학습 정보 및 해당 기기의 계정별 프로필·학습 캐시를 삭제합니다. 게스트 기록 병합 영수증에서는 계정 ID를 즉시 분리하며, 같은 기록의 중복 보상을 막는 무작위 UUID·리비전·생성 시각만 계정과 연결할 수 없는 표식으로 남깁니다.
  • 리마인더·사운드·햅틱·동작 줄이기 같은 기기 설정은 계정 삭제 후에도 남을 수 있습니다. iOS에서는 앱 삭제로 제거됩니다. macOS에서는 앱을 휴지통으로 옮겨도 설정과 Application Support 데이터가 남을 수 있습니다.
  • 열람·정정·삭제·처리정지·동의 철회 요청은 ubergriden@gmail.com으로 접수할 수 있습니다. 요청자 본인 또는 정당한 법정대리인인지 확인할 수 있습니다.

7. 안전성 확보 조치

전송 구간 암호화, 공개키와 관리자 키의 분리, 계정별 행 수준 접근 통제(RLS), 본인 계정 확인이 포함된 동기화 함수, 짧은 수명의 로그인 토큰과 계정 삭제 직전 현재 비밀번호 재확인을 적용합니다. 앱에는 서버 관리자 권한 키를 포함하지 않습니다.
